What is a JavaScript Engine?

JavaScript Engine is a program inside the browser that executes JavaScript code. Like in chrome there is a V8 engine (JS engine) that executes the JS code.
A JavaScript Engine is a program that reads, compiles, and executes JavaScript code. Different environments use different engines. For example, Chrome and Node.js use Google's V8 Engine, while Firefox uses SpiderMonkey.

What is an Execution Context?

Execution context is a workspace where JavaScript code is executed. It is a workspace that JS creates before executing any code. It stores variables, functions etc.
An Execution Context is the environment or workspace where JavaScript executes code. It stores variables, functions, and information about the current execution. JavaScript creates one global execution context when a program starts and creates a new execution context every time a function is invoked.

Why does JavaScript have two phases?

Because both phases have its own unique working and behavior. the first one is the memory creation phase, which allocates a memory space for all the variables and the second phase is the execution phase, in which the JS executes the code using JS Engine (V8) along with it's execution context.

What happens during the Memory Creation Phase?

In the Memory Creation Phase, The JS allocates a memory space for all the variables and functions.

What happens during the Execution Phase?

In the Execution Phase, The JS executes the code using JS Engine (V8) along with it's execution context. Assigns values to variables, calls functions and executes them.

What is undefined?

It is happened when a variable is declared but not initialized. Or in other words, it is a varibale is exist, but it's value has not been assigned yet.

What is the Temporal Dead Zone?

The TDZ is a temporary phase or area in which the variable is exists but not initialized yet. It's a temporary waiting period for a variable to be assigned a value or to be initialized.

What is hoisting?

Hoisting is a concept that allows variables and functions to be declared at the top level conceptually. But in reality it is not, and the JS already allocates memory for them during the memory creation phase.

Why do function declarations work before they appear in the code?

Because during the Memory Creation Phase, the JS already stored the function in the memory and it is ready to be executed and to be called before it appears.

What is the Call Stack?

The call stack is a data structure that keeps track of function calls in JavaScript. When a function is called, it is added to the top of the stack, and when it completes, it is removed from the top of the stack.
It follows the LIFO (Last In, First Out) principle.

What is scope?

Scope refers to the accessibility of varibales and functions. The JS looks to be variable whether it is declared or initialized whereever in the workspace or in execution context.
Scope defines where variables and functions are accessible in a program. JavaScript has global scope, function scope, and block scope.

How does JavaScript find a variable?

JavaScript looks for a variable in the currect scope first. i.e. Local scope (inside a function). If it is found so that's good. If not, then it looks or search it globally in the whole program or execution context.
