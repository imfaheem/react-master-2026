What is Hooks?

Hooks are special React functions introduced in React 16.8, that allow Functional Components to access React features such as state, side effects, context, refs, and other React capabilities.

Why were Hooks introduced?

Hooks were introduced to solve several problems with class components. They make it easier to reuse stateful logic between components using custom hooks, reduce the complexity of class components, eliminate the need for "this" binding, and make React code easier to read, maintain, and test.

Why do Hook names start with use?

Hook names start with use because React and the ESLint Rules of Hooks recognize functions with this prefix as Hooks. This helps enforce the Rules of Hooks and also makes it clear to developers that the function uses React Hook features.

Can Hooks be used inside class components?

No, Hooks cannot be used inside class components. It can only be used and access inside Functional components or inside a Custom Hook.

What is useEffect?

useEffect is a React Hook that runs after React renders the component and updated the DOM. It is used to perform side effects such as fetching data, interacting with browser APIs, starting timers, subscribing to events, or cleaning up resources.

What is a side effect?

A side effect is any operation that interacts with something outside React's rendering process. Such as APIs calls, localStorage, timers, event listeners, or manually updating the DOM.

What is the dependency array?

The dependency array is the second argument of useEffect. It tells React which values to watch and when to re-run the effect.

What is a Cleanup Function?

A Cleanup Function is a function returned from useEffect that React executes before the component unmounts or before the effect runs again due to dependency changes.

When does the Cleanup Function run?

It runs, 1. Before the component unmounts. and 2. Before re-running the effect when dependencies change.

What is a Memory Leak in React?

A memory leak is a situation where a component is unmounted or removed, but the resources it created are still running or are not properly cleaned up. e.g. timers (setInterval, setTimeout), event listeners, subscriptions, and other background tasks. This leads to unnecessary memory usage and performance issues. In React, we prevent memory leaks by returning a cleanup function from useEffect, where we clear timers, remove event listeners, unsubscribe from subscriptions, or clean up other resources.

What is Browser Storage?

Browser Storage is a feature provided by web browsers that allows websites to store data on the user's browser. It helps save information such as user preferences, authentication data, and application state without storing it on the server.
There are two main types of browser storage: 1. Local Storage and 2. Session Storage

What is Local Storage?

Local storage is a browser storage mechanism that stores data permanently in the user's browser. The stored data remains available even if:
1. The page refreshed or reloaded.
2. The browser is closed and opened again.
3. The computer system is restarted.
The data stays in Local Storage until it is explicitly removed by the application, the user clears the browser storage, or JavaScript removes it.

OR

Local Storage is a browser-based storage that persists data even after page reloads, browser restarts, and system restarts, until it is manually removed.
