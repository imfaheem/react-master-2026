What is a State? State is data that belongs to a component and can change over time. Whenever state changes, React automatically re-renders and updates the UI. Why can't we use normal variables instead of state? We cannot use normal variables instead of state because React does not track changes to normal variables. Updating a normal variable does not trigger a re-render, so the UI remains unchanged. However, when state is updated, React detects the change, re-renders the component, and updates the UI. What happens when state changes? When the state changes, React detects the state update and re-runs the component function. A new JSX tree is created. React compares it with the previous JSX. Finally, React updates only the changed parts of the DOM, making the UI update efficiently. Does React re-render the whole webpage? No, React does not re-render the whole webpage. It re-runs the affected component function, compares the new JSX with the previous JSX, and updates only the changed parts of the DOM. What is useState? useState is a built-in React Hook. A Hook is a special function provided by React that allows functional components to store or manage state. What does useState returns? It returns an array which contains the current state value and a setter function used to update that state. What is batching in React? Batching is the process in which React groups multiple state updates into a single re-render to improve performance. What is Rendering? Rendering is the process in which React creates the UI initially and updates it whenever state, props, or context change. What is Virtual DOM? The Virtual DOM is a lightweight JavaScript representation (or copy) of the Real DOM that React uses to efficiently detect changes before updating the browser's DOM. In simple words, Virtual DOM is a JavaScript object that represents the UI. Difference between Real DOM and Virtual DOM? Real DOM is the actual browser DOM that displays the UI, while the Virtual DOM is a light weight representation of the Real DOM. React updates the Virtual DOM first and compares it with the previous Virtual DOM (Diffing algorithm), and then updates only the changed part of the Real DOM (Reconcilation). What is Diffing Algorithm? It is the process in which React compares the new Virtual DOM with the previous Virtual DOM to identify the differences between them. What is Reconciliation? It is the process in which React updates only the changed parts of the Real DOM based on the differences identified during Diffing. Difference between Diffing and Reconciliation? Diffing identifies the differences between the new Virtual DOM and the previous Virtual DOM. Whereas, Reconciliation applies those changes to the Real DOM. What is a controlled component? A form element whose value is controlled by React State instead of DOM. Why are controlled components preferred? Because React becomes the single source of truth, making validation, conditional rendering, and submission logic much easier. What does Single source of truth means? It means that there is only one place from where a piece of data is managed. In controlled components, that place should be React state. So that the UI always stays sync with it.
