Explain in your own words why objects are used instead of many separate variables? Objects are used because they keep related data together in one place. This makes the code simpler, cleaner, and easier to read, update, and maintain than using many separate variables. What is a method? A method is a function that is stored as a property of an object. What does this refer to? this refers to the object that calls the function. Its value depends on how the function is invoked, not where it is defined. Why doesn't this work in arrow functions as object methods? Because arrow functions do not have its own this. they inherit it from the surrounding lexical scope. What is a prototype? A prototype is an object that every JavaScript object is internally linked to. If a property or method is not found on the object itself, JavaScript looks for it in its prototype. Why are prototypes used? Prototypes are used to share common methods and properties among multiple objects. This saves memory because only one copy of the method exists in the prototype instead of every object having its own copy. Where does JavaScript look if a property isn't found on an object? It looks to the prototype object if a propery is not found on an object. JS continues searching up the prototype chain until it reaches null. What is the Prototype Chain? Prototype chain is a mechanism JS used in which each prototype is linked to another prototype for the purpose of searching, or performing any other function. Every object is linked to a prototype which is further linked to another prototype unless it reaches to null. What is Object.prototype? The Objects we create are linked with the Object.prototype to perform multiple functionalities. Why can every object use methods like toString()? Because in the object.prototype there is already a built-in methods associated with it. toString is one of it. What is a JavaScript module? We use modules to avoid a huge single file with 1000s of lines of code. Plus to make the JS code cleaner, reusable and easier to maintain. Why do we use try...catch? try-catch is an error handling syntax in JS in which we write the code in the try block and if any error occurs then the catch block will run and handle the error but the program will run till the end and not break after the error occurs. What is throw? throw is used to manually generate an error in JavaScript. It allows us to create custom errors when a specific condition occurs. When does finally execute? finally block will always execute in the error handling machanism used in JS. It will always execute whether an error occurs or not.
