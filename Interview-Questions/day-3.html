Explain in your own words why objects are used instead of many separate variables?

Objects are used because they keep related data together in one place. This makes the code simpler, cleaner, and easier to read, update, and maintain than using many separate variables.

What is a method?

A method is a function that is stored as a property of an object.

What does this refer to?

This refers to the object that calls the function. Its value depends on how the function is invoked, not where it is defined.

Why doesn't this work in arrow functions as object methods?

Because arrow functions do not have their own this. They inherit it from the surrounding lexical scope.

What is a prototype?

A prototype is an object that every JavaScript object is internally linked to. If a property or method is not found on the object itself, JavaScript looks for it in its prototype.

Why are prototypes used?

Prototypes are used to share common methods and properties among multiple objects. This saves memory because only one copy of the method exists in the prototype instead of every object having its own copy.

Where does JavaScript look if a property isn't found on an object?

It looks in the prototype object if a property is not found on an object. JavaScript continues searching up the prototype chain until it reaches null.

What is the Prototype Chain?

The prototype chain is a mechanism JavaScript uses in which each prototype is linked to another prototype for the purpose of searching or performing other functions. Every object is linked to a prototype, which is further linked to another prototype unless it reaches null.

What is Object.prototype?

The objects we create are linked with Object.prototype to perform multiple functionalities.

Why can every object use methods like toString()?

Because in Object.prototype there are already built-in methods associated with it. toString is one of them.

What is a JavaScript module?

A module is simply a JavaScript file used to split code into multiple files instead of writing everything in a single file.

What is the difference between named export and default export?

Named exports are used to export variables and functions, and they are imported using curly brackets. There can be multiple named exports in a single file. Default export is used only once per file and is imported without curly brackets.

Why do we use modules?

We use modules to avoid a huge single file with thousands of lines of code. They also make JavaScript code cleaner, reusable, and easier to maintain.

Why do we use try...catch?

Try-catch is an error-handling syntax in JavaScript in which we write code in the try block, and if any error occurs, the catch block runs and handles the error without breaking the program completely.

What is throw?

Throw is used to manually generate an error in JavaScript. It allows us to create custom errors when a specific condition occurs.

When does finally execute?

The finally block always executes in JavaScript error handling. It runs whether an error occurs or not.

What is Optional Chaining?

Optional Chaining (?.) is a JavaScript operator used to safely access object properties or methods. If a property is null or undefined, it returns undefined instead of throwing an error.

What is Nullish Coalescing?

Nullish Coalescing (??) is used to provide a default value only if the left side is null or undefined.

What is the difference between || and ??

|| returns the right-hand value if the left-hand value is falsy (0, "", null, undefined). Meanwhile, ?? returns a default right-hand value only when the left-hand value is null or undefined.
